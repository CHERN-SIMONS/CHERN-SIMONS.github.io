对象引用与变量

python将所有数据存为内存对象；而变量是指向内存对象的引用；变量名本身也是对象，也存在内存中，只是一个指针指向于数据。

例如

只是将name指向Tom，第二个就是重新将指针指向Jerry，但是Tom此时没有被销毁，还在内存中。

动态类型：在任何时候，只要需要，某个对象引用都可以重新引用一个不同的对象（可以是不同的数据类型）；内置函数**type()**返回给定数据的数据类型。

因而，在Python中，= 就是将变量名与内存中的某对象绑定：如果对象事先存在，就直接绑定，否则由 "=“ 直接创建引用的对象。

最后连个地址都是同一个，说明Tom已经存在内存中了，我们直接引用它。

变量

保留字：系统本身的命令

命令惯例：1. 单一下划线 _x 的变量名不会被 from module import * 导入

  1. 前后有下划线的变量 _x_ 是系统定义的变量名
  2. 一两个下划线开头，但是结尾没有下划线的变量名 __x 是类的本地变量
  3. 交互模式下，变量名 _ 用于保留最后表达式的结果，类似于mathematica中的%。

注意：变量名没有类型，对象才有。

整体语法格式

主程序：无论当前模块是被别的模块都如还是作为脚本直接执行，都会执行这部分代码

注意：所有模块都有能力执行代码

  1. 最高级别的python语句(没有缩进的)在模块被导入时就会执行，无论是否真的需要执行
  2. 妥当的做法：除了那些真正需要执行的代码之外，所有的功能代码都通过函数建立，因此仅仅才主程序模块中编写大量的顶级可执行代码；用于被导入的模块只应该存在较少的顶级执行代码

每个模块都有一个名为_name_ 的内置变量，此变量值会更具调用次模块的方式发生变化：

  1. 如果此文件被作为模块导入，则__name__的值为模块名称
  2. 如果此文件被直接执行，则 _name_ 的值 ___ name__

6. Python对象特性

Python对象的相关术语

Python程序中保存的所有数据都是围绕对象这个概念展开的：

  1. 程序中存储的所有数据都是对象
  2. 每一个对象都有一个身份、一个类型和一个值；例如，school = "Linux"会一=以"Linux"创建一个字符串对象，其身份是一个指向它在内存中所处位置的指针（其在内存中的地址），而school就是引用这个具体位置的名字
  3. 对象的类型也称对象的类别，用于描述对象的内部表示以及它支持的方法和操作
  4. 创建特定类型的对象时，有时也将该对象称之为该类型的实例
  5. 实例被创建后，其身份和类型就不可改变；如对象值可以修改的，则称之为可变对象，如果对象的值不可修改，则称之为不可变对象
  6. 如果某个对象包含对其他对象的引用，则将其称之为容器
  7. 大多数对象都拥有大量特有的数据属性和方法；属性：与对象相关的值；方法：被调用时将在对象上执行某些操作的函数；使用点 ( . ) 运算可以访问属性和方法

python中两个对象比较涉及：

  1. 值比较：对象中的数据是否相同
  2. 身份比较：两个变量名引用的是否为同一对象
  3. 类型比较：两个对象的类型是否相同

is :运算符用于比较两个对象的身份

type()：用于返回类型的类型

id：返回一个对象的分身，集该对象在内存中的位置

对象类型本身也是一个对象，称为对象的类：该对象的定义为宜，且对于某种类型的所有实例都是相同的；所有类型对象都有一个指定的名称，可用于执行类型检查，如list、dict

数字操作：加 x+y，减x-y， 除法x/y， 截断出发x//y，乘方 x**y, 取模(mod) x%y，一元减法-x，一元加法+x；

序列类型

字符类型：

字符串字面值：把文本放入单引号、双引号或者三引号中；三引号可以进行跨行定义

适用于所有序列的操作和方法：

s[i] :索引运算符

s[i:j] :切片运算符，

s[i:j:stride] ：扩展的切片运算，切片注意负数，负数stride表示从后往前的步数，复数索引表示从最后往前数第几个

len(s): 返回s的元素个数

all(s): 检查s中的所有项是否为True

any(s): 检查s中的任意项是否为True

min(s)和max(s)只是适用于能够对元素排序的序列

sum(s[, initial]) 求和，只是适用于数字序列

适用于可变序列的操作和方法：

s[i] = v 项目赋值

s[i:j] = t 切片赋值

s[i:j:stride] = t 扩展切片赋值

del s[i] 项目删除

del s[i:j] 切片删除

del s[i:j:stride] 扩展切片删除

字符串操作：

s.captitalize() 首字符大写

s.index(sub[, start [,end]]) 找到指定子字符串sub首次出现的位置，否则抛出异常

s.join(t) 使用s作为分隔符链接序列t中的字符串

s.lower() 转换为小写形式

s.replace(old, new [,maxreprelace]) 替换一个子字符串

s.split([sep [,maxsplit]]) 使用sep作为分隔符对一个字符串进行划分，maxsplit是划分的最大次数

s.strip([char]) 删除char开头和结尾的空白或字符

s.upper() 将一个字符串转换为大些形式

Python迭代器和列表解析

迭代：重复做一件事

iterable(可迭代)对象：

  1. 支持每次返回自己所包含的一个成员的对象，所以本身就包含可遍历性质
  2. 对象实现了__iter__方法
    1. 序列对象，如list, str, tuple
    2. 非序列对象，如dict, file
    3. 用户自己定义的一些包含了__iter__()或__getitem__()方法的类

可以用dir()查看有没有__iter__ 方法

迭代器（iterator）又称为游标（cursor），它是程序设计的软件设计模式，是一种可在容器物件（container，如列表等）上实现元素遍历的接口

  1. 迭代器是一种特殊的数据结构，在python中，它以对象的形式存在的，是遵循迭代协议的对象

  2. 可以简单地将迭代器认为就是：对一个集合中的元素，想要执行遍历，那么针对这样的集合的迭代器定义了遍历集体中每一个元素的顺序或者方法

  3. 当结束遍历之后只能通过新建一个可迭代变量，再进行遍历。

  4. 方法__iter__ 或者函数iter，都能实现生成迭代器

  5. 若要在类中实现迭代器，就要定义__next__()方法

  6. for循环可用于任何可迭代对象

    1. for循环开始时，会通过迭代协议传递给__iter__()内置函数，从而能够从可迭代对象中获取一个迭代器，返回的对象含有需要的__next__()方法

Python生成器

背景：对于列表的一般使用或者for循环的遍历，这些操作每次只对一个元素进行操作，所以直接生成一个占内存极大的列表是极为不合理的，这样生成器的需求就出现了

生成器表达式并不真正地创建数字列表，而是返回一个生成器对象，此对象在每一次计算出一个条目之后，把这个条目"产生"（yield）出来

  1. 生成器表达式使用了"惰性计算"或称作"延迟求值"的机制

序列过长，并且每次只需要获取一个元素时，应当考虑使用生成器表达式而不是列表解析
